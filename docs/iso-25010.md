## Défauts détectés et corrections

### 1. Écrasement des comptes locaux

- Défaut : un nouveau compte pouvait remplacer un compte précédent.
- Correction : stockage de plusieurs identifiants locaux et recherche par email.

### 2. Tâches partagées entre comptes

- Défaut : les tâches n'étaient pas séparées par utilisateur.
- Correction : box Hive dédiée à chaque identifiant utilisateur.

### 3. Déconnexion avec tâche active

- Défaut : le statut pouvait rester incohérent après retour sur le compte.
- Correction : boîte de dialogue de confirmation et mise en pause avant déconnexion.

## Vérification par critère

- Sécurité : accès local, pas d'exposition réseau de données de tâche.
- Performance : lecture et écriture locales rapides.
- Maintenabilité : couches séparées et tests unitaires.
- Fiabilité : reprise de session et persistance Hive.
- Utilisabilité : actions visibles, libellés simples.
- Compatibilité : exécution sur les plateformes Flutter disponibles et via Docker.
- Portabilité : code Dart et stockage abstrait.
