# Outils qualité

## 1. GitHub

- Hébergement du code source.
- Suivi des commits réguliers.
- Gestion de version et comparaison des changements.

## 2. SonarQube

- Utilisation attendue pour l'analyse statique.
- Contrôle des duplications, dette technique et règles de qualité.
- Rapport à générer avant validation finale.

## 3. OWASP ZAP

- Utilisation attendue pour un scan de sécurité basique.
- Vérification des surfaces d'entrée et des écrans exposés.
- Rapport à joindre au dossier final.

## 4. Apache JMeter

- Utilisation attendue pour tester les temps de réponse.
- Mesure des scénarios de consultation et sauvegarde.
- Rapport à joindre au dossier final.

## 5. GLPI ou Zammad

- Utilisation attendue pour le suivi des anomalies.
- Gestion des tickets d'incident, problème et changement.
- Un journal simplifié est fourni dans [Suivi des tickets qualité](suivi-tickets.md).

## 6. Statut dans ce projet

- GitHub : utilisé.
- SonarQube : documenté pour intégration.
- OWASP ZAP : documenté pour intégration.
- JMeter : documenté pour intégration.
- GLPI/Zammad : représenté par un suivi de tickets simple dans le dossier qualité.
